# AIOps Prompt Engineering Labs

Repository created during my postgraduate studies in AIOps (AI for IT Operations) and AI for Cloud Engineering.

This project contains practical exercises focused on:
- Prompt Engineering
- Context Engineering
- AI-assisted Infrastructure
- Cloud & Kubernetes Operations
- SRE Runbooks
- Terraform & Infrastructure as Code
- Incident Analysis
- Production-grade Automation

The scenarios are based on the fictional company "Hill Valley Tech" and simulate realistic situations involving:
- Kubernetes
- AWS
- PostgreSQL
- Observability
- Terraform
- FinOps
- Incident Response
- Platform Engineering

## Repository Structure

| Directory | Description |
|---|---|
| q01-dockerfile-lift | Production-ready Dockerfile generation |
| q02-ledger-backup | PostgreSQL backup automation |
| q03-cloud-cost-report | AWS cost optimization analysis |
| q04-ledger-sql-report | SQL analytics generation |
| q05-chronos-modernization | Kubernetes deployment modernization |
| q06-terraform-module | Terraform module generation |
| q07-runbook-chronos | SRE operational runbook |
| q08-postmortem | Production incident postmortem |

## Models Used

## Objective

The goal of this repository is to:
- practice structured Prompt Engineering frameworks;
- compare prompting strategies;
- evaluate AI-generated outputs critically;
- build reusable technical knowledge and portfolio artifacts.
