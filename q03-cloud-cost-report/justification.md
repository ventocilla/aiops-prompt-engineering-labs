# Justification

The T-A-G (Task, Action, Goal) framework was applied by structuring the prompt around a clear operational objective, explicit analytical actions and a measurable business outcome.

## Task

The task section defines the primary responsibility of the model:
analyze the AWS monthly cost CSV and generate a cloud cost optimization report for Hill Valley Tech.

This establishes the core analytical scope of the exercise and frames the model as operating in a realistic FinOps and cloud governance scenario.

## Action

The action section explicitly instructs the model to:
- identify optimization opportunities;
- prioritize recommendations;
- estimate savings;
- evaluate operational risks;
- classify implementation effort;
- define prerequisites.

These actions guide the model toward producing a structured and operationally realistic FinOps assessment instead of a generic cost reduction summary.

## Goal

The goal section defines the business objective:
reduce cloud costs by at least 15% in the next quarter without degrading SLA.

This component is critical because it constrains the recommendations to realistic production-safe optimizations instead of aggressive cost-cutting measures that could negatively impact platform reliability.

The goal also aligns the output with executive-level decision-making by balancing financial optimization and operational stability.
