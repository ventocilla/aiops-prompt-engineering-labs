# Prompt Engineering Frameworks Summary

This repository applies five Prompt Engineering frameworks studied during the postgraduate module.

## Frameworks

| Framework | Meaning | Main Use Cases |
|---|---|---|
| R-T-F | Role, Task, Format | Direct infrastructure/code generation |
| T-A-G | Task, Action, Goal | SLA/KPI-oriented analysis |
| B-A-B | Before, After, Bridge | Modernization and transformation |
| C-A-R-E | Context, Action, Result, Example | Standardization and reusable patterns |
| R-I-S-E | Role, Input, Steps, Expectation | Procedural runbooks and operations |

## Practical Goal

The exercises aim to evaluate:
- prompt structure quality
- output consistency
- framework adherence
- operational realism
- AI-assisted engineering workflows

## Observations

Choosing the correct framework significantly impacts:
- output quality
- response consistency
- operational usefulness
- reasoning depth
- implementation practicality
